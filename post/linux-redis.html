Redis的安装

Redis 是完全开源的，遵守 BSD 协议，是一个高性能的 key-value 数据库。下面详细记录在Linux服务器下安装Redis的过程。

# 1. Redis下载

Linux上下载Reids的方式有很多种，可以通过电脑下载好压缩包上传至Linux服务器，也可以直接通过wget命令直接在Liunx服务器上进行下载。本次安装的是redis官方最新版本，若需其它版本 可以进入 Redis官网 (opens new window)查看其他版本。

wget http://download.redis.io/releases/redis-6.0.6.tar.gz
1

# 2. 安装准备

# 2.1 解压并安装

解压后到解压根目录下的src中执行安装命令。

tar -zxvf redis-6.0.6.tar.gz && make
1

# 3. 运行服务

等着编译完成后，执行ls 命令，会看到src目录下多了些启动文件，进入src目录，执行启动命令启动服务。

./redis-server
1

# 4. 其他配置

# 4.1 后台启动

通过上述步骤，已经可以实现Reids的正常运行，但是在我们关闭了命令窗口或者退出了服务器，服务就自动关闭。这样在实际情况下是不行的，必须保持后台运行，对此我们要进行如下操作，修改redis.conf文件。

vim redis.conf
1

将daemonize no改为=>daemonize yes操作完之后，就重新执行启动命令启动服务即可。

# 4.3 配置redis-cli指令

将redis-cli,redis-server拷贝到bin下，让redis-cli指令可以在任意目录下直接使用

cp /usr/local/redis/bin/redis-server /usr/local/bin/

cp /usr/local/redis/bin/redis-cli /usr/local/bin/
1

# 4.2 设置redis密码

redis-cli

#查看密码
config get requirepass

#设置密码
config set requirepass yourpassword

# 4.3 外网访问redis

首先进行配置防火墙,开放6379端口

firewall-cmd --zone=public --add-port=6379/tcp --permanent
1

重启防火墙以使配置即时生效

systemctl restart firewalld
1

查看系统所有开放的端口

firewall-cmd --zone=public --list-ports
1

# 4.4 可能会遇到的问题

可能开放端口后依旧无法访问，原因是虽然防火墙开放了6379端口，但是外网还是无法访问的，因为redis监听的是127.0.0.1:6379，并不监听外网的请求。这时候需要这样做。

  • 把文件夹目录里的redis.conf配置文件里的bind 127.0.0.1前面加#注释掉
  • redis-cli连接到redis后，通过 config get daemonize和config get protected-mode 是不是都为no，如果不是，就用config set 配置名 属性 改为no。

# 5. 常用命令

#启动
./redis/src/redis-server redis.conf

#停止
kill -9 pid值  或者 ./redis-cli shutdown

#查看服务
ps aux|grep redis

#客户端连接
./redis-cli -h 127.0.0.1 -p 6379

#检测6379端口是否在监听
netstat -lntp | grep 6379
